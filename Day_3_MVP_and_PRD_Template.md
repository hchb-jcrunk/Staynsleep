# Day 3 – MVP & PRD Template

## Part 1: Define Your MVP

### Deliverable Value
- Define the program – points system (assumed single definition)  
- Build infrastructure to collect and accrue points  
- Earn/assign points on user travel  
- Integrate rewards points into existing tools  

**Core Value Proposition:**  
Rewards redemption for travel/trips — financial reductions or benefits  

### Important Features
- Earning credits  
- Balance display  
- Apply credits to travel bookings  
- Communications/emails  
- Early adopter badge  
- Reminders, progress bar for redemption milestones  
- Partner integration for rewards redemption  

### Remove Non-Core
- Remove features not supporting core value prop  
- Explicitly document exclusions and rationale  

### Open Issues
- Exclusions  
- Terms/conditions  
- Analytics  
- Roadmap  

### Q&A Topics
- Early adopters  
- Definition of success/metrics  
- Tiers or levels of rewards  
- Vendor ecosystem

## Part 2: Product Definition

### Overview
- Establishes validation and business case for loyalty rewards feature.  
- Context: how it supports company goals and user experience alignment.  

### Product
- Assess current state, performance, and improvement opportunities.  

### Competitive
- Evaluate how competitors handle rewards and travel incentives.  

### Users
- Identify motivations and feedback for users and non-users.  

### Metrics
- Adoption rate (% of eligible users)  
- Frequency of feature use  
- Revenue share of users utilizing rewards  

### Analysis
- Define whether this feature solves a real problem worth addressing.  
- Explore potential benefits and opportunities created.  

### Objectives
- Explicit project goals and expected outcomes.  

### Success Metrics
- Usage metrics showing value to users  
- Tie to product/company-level metrics  
- Establish baseline and target  

### Messaging
- Define how the feature will be presented to customers  

### Timeline & Release Planning
- Design + engineering estimates and key dates  

### Personas
- Identify personas this feature serves  

### Use Cases
- Narrative per persona describing the journey from problem → solution
